Getting Started

1. Select your serial port and baud rate in the header bar.

2. Click Connect.

3. Paste or type your bash script in the editor (left panel), or drag & drop a .sh file.

4. Click ▶ Send & Run.

5. Watch the output in the terminal (right panel).

You can also type single commands in the $ bar at the bottom of the terminal.

Hex Display

Click HEX to toggle hex dump view. Shows raw bytes like hexdump -C:

00000000 68 65 6c 6c 6f 0a |hello·|

Colors: offset · hex bytes · printable ASCII · CR/LF · non-printable ·

Switching between text and hex preserves data (64KB buffer).

Connection Status

● ConnectedSerial port open, ready to send.
● ReconnectingCable disconnected, retrying automatically.
● DisconnectedNot connected. Select port and click Connect.

Auto-reconnect is enabled by default. If the USB cable is yanked, the app will retry every 3 seconds.

Safety Guard

These commands trigger a confirmation prompt before executing:

rm -rf / · mkfs · dd if= · shutdown · reboot · halt

Disable this in advanced Settings → Behavior → dangerous_cmd_guard.

Mobile / LAN Access

The server binds to 0.0.0.0 by default, making it accessible from any device on the same network.

To use from your phone/tablet:

1. Connect your phone to the same WiFi as this computer.

2. Open the LAN URL shown in the header (or scan the QR code printed in the terminal).

3. Use the full UI — send scripts, view output, run commands — all from your phone.

Flags:

--localRestrict to localhost only (no LAN access)
--host IPBind to a specific interface
--port NChange port (default: 8000)
--no-browserDon't auto-open browser on start

Supported: iOS Safari, Android Chrome, any modern browser. The UI is fully responsive.

Troubleshooting

No ports found?
Check cable, install drivers (CH340, CP2102, FTDI). On Linux: sudo usermod -aG dialout $USER then re-login.

Garbled output?
Verify baud rate matches the board. Increase line delay to 100-200ms. Check flow control and parity settings.

Script fails?
Add #!/bin/bash shebang. Try Send Only, then run manually. Check path is writable.

Connection drops?
Auto-reconnect should recover. Try a different cable (data, not charge-only) or USB port.

Windows: access denied?
Close PuTTY or any other app using the same COM port.
